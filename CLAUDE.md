# CLAUDE.md · 《法术工坊 Spellforge》项目宪法

> 你（Claude Code）每次开工自动读到这份文件。它是压缩版总纲；**完整铁律、当前状态、版本详注以仓库内 `交接说明.gb` 与 `待办.gb` 为准，开工先读这两份**。

## 项目一句话
单文件 HTML/Canvas2D/vanilla-JS 竖版竞技场肉鸽，山海经主题。全部游戏 = 根目录 `index.html`（~12700 行，IIFE，无框架、无构建步骤）。主平台 **iPhone 微信内置浏览器竖屏**，兼容 PC/Android/HarmonyOS。线上 = GitHub Pages（本仓库 main 分支即生产环境）。排行榜 = Supabase（**配置永久冻结，绝不改动**；云端网络受限时排行榜请求失败属预期噪音，不算错误）。

## 角色分工
- **Louie**：产品负责人。只说简体中文、不写代码、多在手机上。真决策给他 **≤4 个可点选项**拍板；实现细节你自主决定、不追问。
- **你**：唯一实现者、测试者、交付者。全程简体中文。

## 云端工作流（每个任务）
1. 读 `CLAUDE.md` → `交接说明.gb` → `待办.gb`，确认当前版本与基线。
2. 在**新分支**上实现。**IMPORTANT: 绝不直接 push main**——main 是线上游戏，合并只由 Louie 在 PR 里点。
3. 过完下方「验证门」再发 PR。
4. **PR 描述全中文**，必须包含：玩家可见变化摘要 / 验证门逐项结果 / 审计行（格式见下）/ 涉及的注册点清单 / 已知接受项。
5. 同一 PR 内同步更新 `待办.gb`、`交接说明.gb`（prepend-only，3 版详注窗口 + 编年史索引 + 归档缓冲区规则见两文件头部说明）。

## 验证门（每次 PR 前全过，缺一不发）
1. JS 语法：整段 script 过 `new Function` 零错。
2. div 平衡：`<div` 与 `</div>` 计数相等（基线随版本变，见 `交接说明.gb` 头部；v104.31 交付时为 948）。
3. CSS 花括号平衡（v104.31 时为 308:308）。
4. Playwright 冒烟：**竖屏 390×844 + 横屏 1280×720 双视口**，真实点击路径（#startBtn → #wandRow 首杖 → 实战 3s），0 个非 CORS/非 Supabase 的 JS 报错。教程浮层用 `localStorage.setItem('sf_tut_seen','1')` 跳过。
5. 0 测试钩子残留：`index.html` 里 `TESTHOOK`、`window.__` 计数必须为 0。**测试钩子只注入临时副本（/tmp），绝不进 index.html。**
6. `APP_NOTIFY_VER === APP_VER`（纯内部/玩家不可见改动除外：不 bump NOTIFY、不写 changelog）。

## 版本与更新日志
- 每次玩家可见改动：同步 bump `APP_VER` + `APP_NOTIFY_VER`，changelog 顶部插条目。
- 青色大版本（`.cl-h.maj`，color:#57e3c4）：新法术/法杖/进化/遗物/Boss/皮肤/系统。金色（无 maj）：修复/平衡/图标/性能/UX。
- 两个青色版本之间的金色小版本折叠成一个区间条（延右端版本+日期，不开新条）。changelog 只写玩家关心的话，不写实现细节。
- `APP_TIME` 用真实执行时间（YYYY-MM-DD HH:MM）。

## 护榜铁律（最高优先级）
已上线的法杖/法术/敌人的 **伤害·冷却·射程 一经发布即冻结**，只允许新增。任何有意的平衡改动必须先给 Louie 选项拍板 + changelog 写明。

## 审计纪律（每个 PR 必执行、必写明）
- **正向审计**：grep 全部关联处一次改齐（显示位/公式/存档续局/牌池/连携/文档/同类配对项）。
- **反向审计**：新增"成族"内容时，先枚举一个已存在同族成员的**全部注册点**，逐点确认新成员到齐（正向 grep 只能找到已加的，找不到漏加的）。法术进化必查 8 点：EVOS/SPELL_DEF/SPELL_TYPE/SPELL_ORDER/ICONS/specialSpellStat/CARD_DEFS evoForm/连携继承。
- **diff 对账**：PR 的 git diff 逐 hunk 归因到本次批次清单，出现不可归因改动=事故信号，查明才可发。
- PR 描述必须含一行：`审计：正向 ✓ / 反向 ✓（对标同族成员 X，逐点确认）`。写不出这行 = 没做 = 不许发 PR。
- **反假阴性**：判断"是否已做"绝不靠一次空 grep 定论——先怀疑查询格式（引号/命名空间/键格式），枚举整个命名空间肉眼比对，能渲染就渲染。
- **硬编码名单脆弱性**：内容分类禁用硬编码 id 名单（新内容会静默漏掉），用属性派生；既有脆弱名单登记册见 `交接说明.gb`，新增内容逐一核对。

## 图标与内容规范
- 一切新内容（法杖/遗物/法术/敌人/Boss/符文…）**必须同时带手绘 SVG 图标**，接进 ICONS 正确命名空间（wand_/relic_/spell_/syn_/…）。**全游戏 100% 手绘 SVG，禁止 emoji 占位。**
- SVG 规格：viewBox 0 0 64 64、不写 width/height、内容在 5–59px 安全区、描边 #241636 粗 2.6–3.2 圆角连接、平涂饱和色 2–4 种、小高光、无渐变/滤镜/外部引用。必须在 #2b1d4a 深紫卡底上 48px 可辨识（用像素色板分解程序化检查，别只靠肉眼）。
- 机制描述（desc）与行为同步更新，山海经文风：简练、有出处感，对齐现有 SPELL_DEF/RELICS 语气。

## 渲染与平台铁律
- **亮地形可读**：一切战斗 UI/特效必须在全 9 地形可读，尤其雪原(#e0e8f0)/沙漠(#d8a050)——source-over 实色主体 + #241636 深描边；'lighter' 叠加在亮底会消失；全屏闪光峰值压低。新特效必截雪原+沙漠图目检。
- **方向弹八向必测**：按速度角旋转的弹体，交付前八方向渲染截图；左行翻转已知解法 `if(vx<0) ctx.scale(1,-1)`（rotate 之后）。轮廓上下对称可免翻转。几何/朝向永远靠渲染验证，不靠心算。
- 跨屏 + 三输入：窄屏≤375/宽屏/横屏/折叠屏不错位；触屏、键盘、手柄三输入互不冲突。
- 变长元素的新 UI 必须可滚动（overflow-y:auto + -webkit-overflow-scrolling:touch + justify-content:safe center）。
- CSS 插入核对 @media 作用域（勿把通用规则塞进横屏专用块）；Playwright 必带竖屏视口。
- iOS/微信已知坑：用 100svh 不用 100dvh；微信移动端无 Gamepad API；HarmonyOS ArkWeb 静默忽略 requestFullscreen；iOS Safari 在 location.replace 后 canvas GPU 纹理降级；`#debug` 哈希开关是远程 iOS 诊断工具。

## 工程习惯
- **归档卷平时禁读**:仓库根目录《Spellforge归档*.gb》一律不在开工/开局时读取或载入(避免污染上下文);仅当确需追查历史版本细节时,按需 `grep` 定位相关段落、只读命中处。
- 大改动拆小步：每步独立 assert（如 Python `rep(a,b)` + `assert s.count(a)==1`），长内容（SVG/文案）先写独立文件再注入，每步后 grep 核对状态。
- 上下文节流：按行号范围读码、grep 计数代替全文粘贴。
- 回合内合并：侦察一次打包、验证一次打包、长输出写文件只回结论（省轮次、少中断）。
